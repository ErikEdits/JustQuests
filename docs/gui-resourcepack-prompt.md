## 1. Project context

JustQuests is a lightweight quest-book mod (NeoForge now; Fabric/Forge and
a Paper plugin later). v0.1 is command-only. v0.2 adds an in-game GUI.

The GUI must feel like a natural, vanilla-quality Minecraft screen — calm,
readable, "Mojang could have shipped this." It is NOT a flashy modern UI;
it's pixel-art that sits beside vanilla containers without clashing.

The community picks the overall layout via a poll, so produce assets for
**both** candidates (they share most widgets):
- **Layout A — Book style:** a two-page open book (the "quest book"
  identity; FTB-Quests-like).
- **Layout B — Single panel:** one modern panel, scrollable list on the
  left + detail pane on the right.

Design decisions this spec implements (from open-questions.md):
- Q3 categories with distinct symbols (AI / custom / datapack / team)
- Q28 locked quests shown as a "???" + lock teaser (opt-in)
- Q43 optional HUD tracker overlay
- Q44 per-quest custom icon (item as icon) with fallback
- Q45 sort/group by category, status, custom order
- Q46 search box that only appears at high quest counts
- Q48 reward claim button (completed-but-unclaimed state)
- Q49 choice rewards (pick 1 of N)
- Q9 difficulty Easy/Normal/Hard indicators
- Q12 toast notifications (poll-gated)
- Q20/Q21 localized text → generous, language-safe text areas

## 2. Hard technical constraints (cross-version: MC 1.14 → newest)

Non-negotiable. The same PNGs must work unchanged from 1.14 to the latest
version; only the mod's drawing code differs per version.

- **Plain PNG atlases only**, drawn via `blit(x,y,u,v,w,h)` from explicit
  coordinates.
- **Do NOT** use the 1.20.2+ `textures/gui/sprites/` system or any
  `.mcmeta` 9-slice/animation metadata — it doesn't exist pre-1.20.2.
  Bake 9-slice regions into the atlas (Section 9); the mod slices in code.
- **Atlas 256x256** (power of two). Need more room? add `quest_gui_2.png`,
  don't grow to 512.
- **Integer coordinates** for every sprite; no sub-pixel placement.
- **≥2 px fully-transparent gutter** between sprites (prevents bleeding).
- **No** shaders, emissive/normal maps, or baked animation.
- **No anti-aliasing.** Hard pixel edges. No smooth gradients. The only
  soft effect allowed is an ordered 50% dither for a single shadow row or
  a translucent panel body, and only where explicitly noted.
- **Design at 1x** (texture px = GUI px). The game scales by the user's
  GUI scale; never pre-scale.
- **Palette-only** colors (Section 3) plus full transparency.

## 3. Art direction & palette

Style: **vanilla-plus**. Start from vanilla's container/book look, add warm
parchment + oak so it reads as a quest book.

Use ONLY these colors. Each ramp is **dark → mid → light** for shading
(light source top-left, shadow bottom-right, 1 px bevel on raised edges,
inverted bevel on pressed/inset elements).

**Neutral grays** (frames, generic panels, slots):
| role | hex |
|---|---|
| outline (near-black border) | `#2B2B2B` |
| deep shadow | `#555555` |
| inset / mid shadow | `#8B8B8B` |
| base gray | `#C6C6C6` |
| highlight | `#FFFFFF` |

**Parchment** (pages, list rows, detail body):
| role | hex |
|---|---|
| parchment shadow | `#B8A06A` |
| parchment shadow-mid | `#D6C492` |
| parchment base | `#E8D8A0` |
| parchment highlight | `#F4ECC6` |

**Oak / leather** (book cover, frames, header bands):
| role | hex |
|---|---|
| oak shadow | `#6E4A24` |
| oak dark | `#8A5A2B` |
| oak mid | `#A8743E` |
| oak light | `#C49A63` |

**Accents** (status & feedback — each with a shadow tone):
| role | base | shadow |
|---|---|---|
| green (complete/success) | `#5ABE50` | `#3C9637` |
| gold (available "!"/highlight) | `#E0B33A` | `#B07E1E` |
| blue (active/info) | `#5B8DD6` | `#3C6196` |
| red (locked/abandon/error) | `#C24B4B` | `#8E3030` |
| purple (AI category) | `#9B6BD6` | `#6E47A0` |

Shading rules:
- Raised element: top + left edge = light tone, bottom + right = shadow,
  1 px `#2B2B2B` outline around the whole shape.
- Inset element (slot, groove, text field): inverted — shadow top-left,
  light bottom-right.
- Buttons get a 1 px highlight inner line on the top edge.

Text is rendered by the mod — **never bake text into textures**. Default
text colors the mod uses: `#404040` on parchment, `#FFFFFF`+shadow on dark
headers, accent colors for status words. Reserve quiet areas for text.

## 9. 9-slice specification (how to bake stretchable sprites)

Buttons, panels, the search box, HUD bg, and toast bg must stretch to any
width/height without distortion. Build each as a 9-slice and let the mod
slice it:
- **Corner size:** 3 px (so a 1px outline + 2px bevel survive).
- Layout each 9-slice sprite as: 4 corners (3x3), 4 edges (top/bottom
  3xN, left/right Nx3), 1 center fill. The mod tiles/stretches edges +
  center, draws corners 1:1.
- Provide the sprite at a representative size (e.g. button 80x20) such
  that the middle is uniform and safely stretchable; mark the corner
  inset (3 px) in TEXTURE_MAP notes so the code knows the slice lines.
- Center fill must be a flat tile (no feature that would smear when
  stretched).

Document, per 9-slice sprite, the corner inset and which axes stretch.

## 17. Global state rules (apply to every interactive sprite)

Every interactive widget needs these visual states; if a sprite table
above doesn't list one, derive it with these rules:
- **normal** — default.
- **hover** — 1px gold (`#E0B33A`) outline OR +1 lightness step.
- **pressed/active** — inverted bevel (shadow moves to top-left).
- **disabled** — desaturated to grays, flat (no bevel), ~60% perceived
  contrast.
- **focused** (keyboard nav, Q65) — 1px blue (`#5B8DD6`) dotted/solid
  outline, distinct from hover.
Provide focused variants at least for buttons and the search box.

## 19. Localization-safe spacing (Q20/Q21)

Text is code-drawn and translated (German, Japanese, etc.), which is often
wider/taller than English. Therefore:
- Make all **text areas generous** — title areas fit ~24 chars, button
  labels fit ~14 chars without the art crowding them.
- Buttons are 9-slice (Section 9) so the mod can widen them for long
  labels — design the middle to tile cleanly.
- Don't put decorative art where text will sit; keep interiors quiet.

---

## 20. Accessibility

- **Contrast:** text-bearing backgrounds must give clear contrast with the
  mod's text colors (`#404040` on parchment ≈ ok; ensure ≥4.5:1 where
  feasible). Don't place mid-gray text zones on mid-gray fills.
- **Colorblind:** every icon distinguishable by shape in grayscale
  (Section 24 verifies this).
- **GUI scale:** must stay readable at GUI scale 1 (small) through 4
  (large) on a 1080p screen — verify at 1x/2x/3x.
- **No meaning by color alone** — pair color with a shape/glyph.

## 21. Generation method (Python + Pillow)

- One readable generator script per atlas in `scripts/`, so coordinates
  are reproducible and the map is generated from the same source.
- Draw at **1x** on an integer grid; scale up only for review mockups
  (NEAREST). Save **32-bit RGBA**, no interlacing, no color profile.
- Emit `TEXTURE_MAP.md` from the script (single source of truth).
- Example skeleton:
```python
from PIL import Image
A = Image.new("RGBA", (256,256), (0,0,0,0))
px = A.load()
PAL = { "outline":(43,43,43,255), "parch":(232,216,160,255),
        "parch_hi":(244,236,198,255), "oak":(168,116,62,255),
        "green":(90,190,80,255) }   # ...full palette...
def rect(x0,y0,x1,y1,c):
    for y in range(y0,y1+1):
        for x in range(x0,x1+1):
            if 0<=x<256 and 0<=y<256: px[x,y]=c
# ...draw each sprite at its allocated (u,v)... record into a list...
A.save("atlases/quest_gui.png")
# write TEXTURE_MAP.md rows from the recorded sprite list
```
- A separate `assemble_mockups.py` may read the finished atlas + map to
  build the review mockups (this also proves the map is correct).

## 24. Acceptance criteria / QA (must all pass)

1. Atlases are exactly 256x256 RGBA; every sprite within 0..255.
2. No two sprite bounding boxes overlap; every sprite has ≥2px gutter
   (assert in the script).
3. Only palette colors used (assert: every opaque pixel ∈ palette, except
   documented dither/alpha rows).
4. Every TEXTURE_MAP entry matches the actual atlas pixels (assert).
5. Every icon is distinguishable in grayscale (save a grayscale contact
   sheet as proof).
6. Mockups are assembled solely from the atlases via the map (no extra
   drawing) — proves the map is usable.
7. Mockups readable at 1x, 2x, 3x.
8. 9-slice sprites have documented corner insets and a flat, tileable
   center.
9. No text baked into any texture.
10. No use of the 1.20.2 sprite system or `.mcmeta` 9-slice/animation.

## 26. What NOT to do

- No 1.20.2+ sprite/mcmeta system; no animations.
- No baked text.
- No off-palette colors, smooth gradients, or anti-aliasing.
- No atlas larger than 256x256 (add another atlas instead).
- No assumptions about or changes to mod code — assets only.
- No meaning conveyed by color alone.
- Don't generate early — this is scheduled for next month's v0.2 work.
